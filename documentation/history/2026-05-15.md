## Fixed "dynamic <script> element creation" review warning

- Symptom: catalog reviewer reported `Found 2 dynamic <script> element creations` against the released `dist/main.js`. Both occurrences lived in `node_modules/setimmediate/setImmediate.js`, bundled transitively via `jszip` (required by `src/app/services/import/rmdoc-import.service.ts`). The two `createElement("script")` calls are the polyfill's old-IE branch — dead code in Electron — but the reviewer flags them regardless.
- Fix: added a `BunPlugin` (`stripSetImmediatePolyfillPlugin`) in `scripts/build.ts` that `onLoad`-intercepts any path matching `*/setimmediate/setImmediate.js` and replaces its contents with a Promise-based stub. The stub keeps the runtime contract (define `setImmediate`/`clearImmediate` on the global if missing) using `Promise.resolve().then(...)` and performs no DOM access.
- Wired the plugin into `Bun.build({ ..., plugins: [stripSetImmediatePolyfillPlugin] })`.
- Verified: `grep "createElement.*script" dist/main.js` returns nothing after rebuild.

## Render full page bounds instead of cropping (issue #3)

- Symptom: pages where the user scrolled while writing on the device (creating strokes beyond the standard 1404×1872 viewport) were rendered at the fixed PAGE_WIDTH × PAGE_HEIGHT size, cropping the lower portion of the content. Confirmed by reporters @toxichavok34-design, @risograph, @Andromedanevel.
- Root cause: `renderPageToCanvas` in `src/app/services/renderer/page-renderer.service.ts` always allocated a 1404×1872 `OffscreenCanvas` and the stroke renderer used a hardcoded `X_OFFSET = PAGE_WIDTH / 2`. Anything outside that rectangle was clipped silently.
- Fix:
    - New pure helper `src/app/services/renderer/stroke-bounds.ts::computeStrokesBounds(strokes)` walks every non-eraser point, expands each by its rendered radius (`point.width * widthMultiplier * stroke.thickness / 2`, clamped to ≥ 0.5px), and returns `{minX, maxX, minY, maxY}` or `null` when no visible content exists.
    - `renderPageToCanvas` now:
        - keeps `PAGE_WIDTH × PAGE_HEIGHT` as a floor so short, in-bounds pages look identical to before.
        - grows the canvas in any direction the strokes overflow (`±halfWidth` symmetric around 0 for X; downward and upward via `ctx.translate(0, topExtra)` for negative Y).
        - adds `EDGE_PADDING = 8` px around grown bounds so stroke tips never touch the image edge.
    - `renderStroke` now accepts an `xOffset` parameter (defaults to `PAGE_WIDTH / 2` for legacy callers) so it can center on whatever canvas width the page-renderer chose.
- Tests: `src/app/services/renderer/stroke-bounds.spec.ts` covers empty/eraser-only/no-points returning null, single-point halo, per-point radius expansion (BallPoint), multi-stroke aggregation, content beyond `PAGE_HEIGHT`, negative Y, and that erasers don't extend bounds. 9 new tests; suite now 133 / 0.
