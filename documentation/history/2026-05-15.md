## Fixed "dynamic <script> element creation" review warning

- Symptom: catalog reviewer reported `Found 2 dynamic <script> element creations` against the released `dist/main.js`. Both occurrences lived in `node_modules/setimmediate/setImmediate.js`, bundled transitively via `jszip` (required by `src/app/services/import/rmdoc-import.service.ts`). The two `createElement("script")` calls are the polyfill's old-IE branch — dead code in Electron — but the reviewer flags them regardless.
- Fix: added a `BunPlugin` (`stripSetImmediatePolyfillPlugin`) in `scripts/build.ts` that `onLoad`-intercepts any path matching `*/setimmediate/setImmediate.js` and replaces its contents with a Promise-based stub. The stub keeps the runtime contract (define `setImmediate`/`clearImmediate` on the global if missing) using `Promise.resolve().then(...)` and performs no DOM access.
- Wired the plugin into `Bun.build({ ..., plugins: [stripSetImmediatePolyfillPlugin] })`.
- Verified: `grep "createElement.*script" dist/main.js` returns nothing after rebuild.

## Foldable folders in the notebook panel (issue #4)

- Symptom: user reported the panel is hard to navigate without a way to collapse folders, and that top-level notebooks were initially invisible because they had no folder header.
- Fix in `src/app/ui/remarkable-panel-view.ts`:
    - Introduced a `collapsedFolders: Set<string>` field on the view (in-memory, per-session). Folder headers are now clickable buttons (with `role=button`, `tabindex=0`, `aria-expanded`, Enter/Space keyboard activation) that toggle membership in the set and re-render.
    - Each folder header now shows a chevron (`chevron-right` collapsed, `chevron-down` expanded), the folder icon, the folder name, and a notebook count badge.
    - Top-level notebooks (previously grouped under a literal `'Root'` string with no header) are now grouped under `ROOT_FOLDER_KEY = '__root__'` and rendered first with the visible label `Top level`. Resolves the "I can't see my toplevel notebooks" half of the issue.
    - When a search query is active, collapsed state is ignored so search matches inside collapsed folders are never hidden.
- Styles in `src/styles.src.css`: added hover, focus-visible outline, chevron sizing, label truncation, and a count-badge pill on `.remarkable-folder-header`.
- README: updated the panel feature bullet to mention "foldable folder hierarchy".
