# 2026-06-27

## OCR transcription: one markdown note per notebook (GP-13)

Added optional OCR of synced reMarkable pages into one assembled markdown note per
notebook, newest page on top. OCR runs via a **local** server (md_capture_server
`/ocr`); no OCR/API keys live in this plugin.

### Settings

- `ocrEnabled` (default `false`) and `mdserverOcrUrl` (default
  `http://localhost:1250/ocr`) added to `PluginSettings` + defaults + `loadSettings`
  merge. New **OCR transcription** settings section (`settings/components/ocr-section.ts`).

### Flow

- Hooked into the existing per-notebook pipeline (`processNotebook`), so it respects
  the GP-12 source-folder/newest scoping (OCR applies to whatever the sync renders).
- Per page: hash the rendered image (`srcHash`). If `srcHash` matches the stored
  per-page state, skip — no OCR call, no rewrite. Otherwise POST the image to
  `mdserverOcrUrl` (multipart `image`), read `{markdown}`, hash it (`ocrHash`).
- Per-page state (`pageId`/`srcHash`/`ocrHash`) persisted in `syncStore`
  (`NotebookSyncState.pages`). `updateState` preserves existing per-page state when a
  caller omits the map (non-OCR syncs never wipe OCR progress), and the pipeline
  passes the map only when non-empty (an all-pages-failed run can't clobber state).

### Assembly (`domain/ocr-markdown.ts`, pure + unit-tested)

- Managed block per page: `<!-- rm:page=<id> src=<h> ocr=<h> -->\n## Page N\n<md>\n<!-- /rm:page=<id> -->`.
- New page → inserted at the top of the managed region (newest topmost). Changed page
  → block replaced in place. Content outside any block is never touched.
- Anti-clobber: if a block's on-disk body no longer hashes to its `ocr=` marker (user
  hand-edited), the fresh block is inserted above and the edited one demoted to a
  collapsed `> [!note]- superseded` callout — never silently overwritten.
- OCR bodies are sanitized (`<!--` → `&lt;!--`) so transcribed text can't forge or
  prematurely close a managed-block marker.
- Fail-soft: a per-page OCR/network error or a markdown-write failure logs and leaves
  the prior block/state intact; it never aborts the sync or wipes the note.

## OCR rate-limiting: paced + resumable OCR (GP-16)

Production bug: the first sync of a large notebook burst-OCR'd every page and tripped
Mistral's rate limit (`HTTP 429: rate_limited` at page 46/50), aborting the whole
notebook. Fixed with serialized, paced, retryable, resumable OCR — no separate queue;
the syncStore is the implicit worklist.

### OCR service (`services/ocr/ocr.service.ts`)

- `ocrPageImage` gained an `OcrRequestOptions` param and an internal retry loop:
  retries `429` / `5xx` with exponential backoff (`base * 2^attempt`, default base 500ms,
  cap 30s, ~5 retries), honoring `Retry-After` (delta-seconds or HTTP-date). Non-retryable
  status / malformed body / cap-exceeded throws so the caller fails soft for that page.
- Pure, unit-tested helpers: `isRetryableStatus`, `parseRetryAfterMs`, `computeBackoffMs`,
  case-insensitive `headerValue`. `sleep`/`now` are injectable for hermetic tests.
- New `utils/sleep.ts` (`window.setTimeout`).

### Pipeline (`services/pipeline/notebook-pipeline.service.ts`)

- Pages OCR'd **serially** (already), now with a configurable inter-request delay
  (`ocrRequestDelayMs`, default 400ms) between requests (none before the first; skipped
  for unchanged/skipped pages).
- **Incremental persist**: each page's block is written into the note and its `ocrHash`
  recorded in syncStore as soon as it succeeds (md write → then syncStore persist; a copy
  `{...nextPages}` is passed so immer's frozen snapshot doesn't freeze the live map).
- **Resume**: the notebook's cloud mtime is held at its prior value until _every_ page is
  OCR'd (`allPagesOcrd`), so a partial/interrupted run stays "needs sync" and the next sync
  re-enters and OCRs only the pages still missing an `ocrHash`. Per-page `srcHash` skip
  prevents re-OCR of done pages; re-folding an existing page replaces its block in place
  (no duplicates).
- Rate-limit/error surfaced as Notices ("rate limited — backing off, retrying page N";
  "OCR failed for page N — will retry on next sync") instead of a hard abort.

### Settings

- `ocrRequestDelayMs` (default 400) added to interface/defaults/loadSettings merge + a
  number field in the OCR settings section. Documented in README + configuration.md.

- `bun run tsc`, `bun run lint` (max-warnings 0), `bun test` (193 pass, +16 new),

## In-app sync-log panel (GP-15)

Motivation: a silent server-side 429 stalled OCR with zero in-app visibility (had to
read docker logs to find it). Added a sidebar view that makes sync + OCR activity —
especially failures with their reason — visible inside Obsidian.

### Pieces

- `domain/sync-log.ts` (pure, unit-tested): `SyncLogEvent`/`SyncLogStatus`, capped
  `appendEvent` (last `MAX_SYNC_LOG_EVENTS` = 200), `statusMarker`, `formatClock`,
  `formatEvent`.
- `services/log/sync-log.service.ts`: in-plugin event bus + capped in-memory buffer —
  `emit`/`getEvents` (returns a snapshot copy)/`clear`/`subscribe`. Each event is also
  mirrored to the dev console via `log()`. Injectable `now()` for hermetic tests.
- `ui/sync-log-view.ts`: `ItemView` (right leaf, type `remarkable-sync-log`,
  "reMarkable sync log"). Subscribes on open, unsubscribes on close, re-renders on each
  event; newest-first rows with ✓/⊘/✗/• markers + a Clear button. CSS in
  `styles.src.css` (Obsidian theme vars; `--text-success`/`--text-error`/etc.).
- Command **Open reMarkable sync log** + a log button on the notebook panel header;
  both call `plugin.activateSyncLogView()`.

### Wiring

- `plugin.syncLogService` created first in `onload` (before any emitter); view registered.
- `syncAllNotebooks` emits run-start (with trigger startup/interval/manual via a new
  `SyncAllOptions.trigger`) + the run summary + list-failure reason.
- `processNotebook` emits per-page OCR outcomes (✓ transcribed / ✗ failed **with the
  error message** / rate-limit "retrying" / render-fail / persist-fail) and a per-notebook
  summary (`N transcribed, M unchanged, K failed`); download/parse/pipeline errors too.
- Panel-initiated syncs emit a manual run-start for parity.
- Scope cuts honored: no persistent log DB, no search/filter UI, no md_capture log
  mirroring — just the plugin's own events.

- `bun run tsc`, `bun run lint` (max-warnings 0), `bun test` (204 pass, +11 new),
  `bun run build`, `bun run format` clean; `bun audit` only the pre-existing dev-only
  js-yaml moderate. High-effort `/code-review` (3 angles): no correctness bugs; applied
  the cheap hardening (onOpen double-subscribe guard, getEvents snapshot copy,
  panel run-start emit) + these docs.
- Hermetic tests: `sync-log.spec.ts` (cap math, markers, clock/format) and
  `sync-log.service.spec.ts` (emit/clear/subscribe/unsubscribe, cap, timestamping).

## OCR figure links → real page image embeds (GP-18)

Bug: Mistral OCR emits `![img-N.jpeg](img-N.jpeg)` links for figures it detects, but
guesses a non-existent relative path, so Obsidian shows "img-0.jpeg could not be
found" (60 broken refs in `handwritten/6.md`). The real full-page image _is_ saved
(`<targetFolder>/<folderPath>/<notebook>/<notebook>-P<NNN>.jpeg`), so we repoint the
refs to it rather than dropping them.

### Pieces

- `domain/ocr-markdown.ts` (pure, unit-tested): `rewriteImagePlaceholders(md, pageImagePath?)`
  replaces the precise Mistral placeholder shape (alt AND target are bare `img-N.<ext>`)
  with a single `![[pageImagePath]]` embed per page (dedup via a one-shot flag);
  user-added links with real targets/custom alts are preserved; with no image the refs
  are dropped (no dangling). `parsePageNumber("Page N")`. `migrateBlocksImagePlaceholders(
content, resolveImagePath)` rewrites each managed block + re-renders it with a fresh
  `ocr=` marker matching the new body, returning the (pageId, new ocrHash) of changed blocks.
- `services/migration/clean-image-placeholders.ts`: one-time migration — indexes saved
  page images by basename (disambiguated by the notebook's own subfolder), scans target
  md files with managed blocks, rewrites + reconciles the stored per-page `ocrHash` (no
  re-OCR) so the anti-clobber guard never demotes a migrated block to "superseded".
  Skips entirely when OCR has never run.
- On write: the pipeline cleans each page's OCR markdown via `rewriteImagePlaceholders`
  using `buildPagePath(...)` for the real image, before hashing/assembling — new syncs
  never produce broken refs.
- Trigger: auto-run once on load (`imgPlaceholderMigrationDone` flag, set only on
  success) before the startup sync; plus a manual **Fix OCR image links in notes** command.

### Decisions

- Anti-clobber reads the FILE's `ocr=` marker vs the file body, so rewriting the block +
  marker is sufficient; the stored syncStore `ocrHash` (currently write-only) is also
  reconciled per the goal.
- Same-named notebooks already collide at the note level (`<name>.md`); GP-18 doesn't
  worsen it. Logged to builder-feedback for the manager.

- `bun run tsc`, `bun run lint` (max-warnings 0), `bun test` (216 pass, +12 new),
  `bun run build`, `bun run format` clean; `bun audit` only the pre-existing dev-only
  js-yaml moderate. High-effort `/code-review` (3 angles): on-write fix verified correct
  on all angles; hardened the migration (skip-when-no-OCR, success-only flag, subfolder
